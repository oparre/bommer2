2. NEW FEATURE: SYNC FROM devapp
===============================================================================

Location: Projects List page (new button next to Create New Project)
Button: "Sync from devapp" (with sync icon)

How it works:
1. Click "Sync from devapp" button
2. System fetches all projects from devapp database (project_management.projects)
3. Modal shows which projects are new vs already imported
4. Select projects to import (all selected by default)
5. Click "Import Selected Projects"
6. Projects are imported with:
   - Generated codes (e.g., GAMING-016, ARCADE-013)
   - Mapped statuses (In Progress → active, Pending → planning)
   - Current user as owner/creator
   - All original data preserved

3. FIELD MAPPING: devapp → bommer
===============================================================================

devapp Field          → bommer Field          Transformation
----------------------------------------------------------------------------
id                    → (used for code gen)   Part of generated code
name                  → name                  Truncated to 200 chars
description           → description           Direct copy
status                → status                Mapped (see below)
start_date            → (not mapped)          Stored in description if needed
end_date              → (not mapped)          Stored in description if needed
created_by_user_id    → created_by/owner_id   Falls back to current user
created_at            → (preserved in audit)  Logged in audit trail
(none)                → code                  Generated (e.g., GAMING-016)
(none)                → priority              Defaults to "medium"
(none)                → is_optional           Defaults to 0 (false)

STATUS MAPPING:
devapp Status    → bommer Status
--------------------------------
Pending          → planning
In Progress      → active
Completed        → completed
On Hold          → on-hold
Cancelled        → cancelled

CODE GENERATION:
Pattern: [CODEBASE]-[ID]
Examples:
- "Gaming Desks" (ID 16) → GAMING-016
- "000561 Mini Arcade Stick" (ID 13) → 000561-013
- "filippo e il panaro" (ID 14) → FILIPP-014

8. KNOWN LIMITATIONS
===============================================================================

1. User ID mismatch:
   - devapp user IDs don't match bommer user IDs
   - All imported projects use current user as owner/creator
   - Solution: Create user mapping table (future enhancement)

2. Code uniqueness:
   - Generated codes may conflict if similar names exist
   - System skips duplicates during import
   - Manual code editing not available after import

3. One-way sync:
   - Changes in bommer don't sync back to devapp
   - Changes in devapp after import require manual re-import

4. Date fields not mapped:
   - start_date and end_date from devapp not stored
   - Could be added to description field if needed
